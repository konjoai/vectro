What is Vectro?

Big AI memory, made small.

Modern AI turns every sentence, image, or document into a long list of numbers — an AI fingerprint that captures what it means, not just the words.

A million of these fingerprints from OpenAI's text-embedding-3-small takes about 6.1 GB of memory. Vectro shrinks that to 1.5 GB with 8-bit compression, or as little as 192 MB with 1-bit compression — and search is just as accurate.

The math runs in Mojo SIMD on Apple Silicon and Linux x86, with NumPy fallbacks everywhere else. Existing LangChain / LlamaIndex / Haystack / DSPy code drops in unchanged — no rewrites, no new vector database to learn.

Why Vectro?

Three places this is a force multiplier.

If your product turns text or images into "meaning" and looks them up later, Vectro probably saves you money, latency, or both.

Products, articles, tracks, jobs — anywhere you compare a user to a catalog. Vectro fits the whole catalog in RAM on a single box, so personalisation doesn't need a Postgres-sized vector cluster.

Code

One line. Same library you already use.

Pick the framework you have. Add compression_profile=. That's it.

LangChainVectorStore
from python.integrations import LangChainVectorStore
from python.embeddings import OpenAIEmbeddings

embed = OpenAIEmbeddings(model="text-embedding-3-small")
store = LangChainVectorStore(
    embedding=embed,
    compression_profile="balanced",   # 4× smaller
)

store.add_texts(["Paris is the capital of France"])
docs = store.similarity_search("capital of France", k=3)
LlamaIndexVectorStore
from python.integrations import LlamaIndexVectorStore

DSPyRetriever
import dspy
from python.integrations import VectroDSPyRetriever
from python.embeddings import OpenAIEmbeddings

rm = VectroDSPyRetriever(
    embed_fn=OpenAIEmbeddings(model="text-embedding-3-small"),
    k=5, compression_profile="balanced",
)
rm.add_texts(["Paris is the capital of France"])

dspy.settings.configure(rm=rm)
prediction = rm("What is the capital of France?")
