What is vectro?

A compression layer for embedding indexes — built for RAG.

A 1M-vector OpenAI text-embedding-3-small index costs 6.1 GB at float32. With Vectro INT8 it's 1.5 GB. With NF4 it's 0.76 GB. With binary it's 192 MB — and recall is preserved well past the practical RAG bar on every profile.

The math runs in Mojo SIMD on Apple Silicon (12M+ vec/s INT8 throughput) with NumPy fallbacks everywhere else. The integration layer is plain Python — your existing LangChain / LlamaIndex / Haystack / DSPy code drops in unchanged.

k-NN, MMR, hybrid search via reciprocal rank fusion, re-ranking, metadata filters, persistent save/load, async everywhere. The parts that matter for production retrieval, with none of the bloat.

Code

Drop-in adapters.

One compression_profile argument — that's the entire learning curve.

LangChainVectorStore
from python.integrations import LangChainVectorStore
from python.embeddings import OpenAIEmbeddings

embed = OpenAIEmbeddings(model="text-embedding-3-small")
store = LangChainVectorStore(
    embedding=embed,
    compression_profile="balanced",   # 4× INT8
)

store.add_texts(["Paris is the capital of France"])
docs = store.similarity_search("capital of France", k=3)

DSPyRetriever
import dspy
from python.integrations import VectroDSPyRetriever
from python.embeddings import OpenAIEmbeddings

rm = VectroDSPyRetriever(
    embed_fn=OpenAIEmbeddings(model="text-embedding-3-small"),
    k=5, compression_profile="balanced",
)
rm.add_texts(["Paris is the capital of France"])

dspy.settings.configure(rm=rm)
prediction = rm("What is the capital of France?")
